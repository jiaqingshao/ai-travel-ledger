## 二、主要竞品深度分析

### 2.1 Splitwise（国际老大）

- **平台**: iOS / Android / Web
- **用户量**: 全球 1000 万+
- **评分**: 4.5
- **下载量**: Google Play 1000 万+

#### 核心功能（Core 免费）
- 添加群组和朋友
- 分摊费用、记录债务
- 均摊 / 不均摊 / 按% / 按份额
- 计算总余额
- 简化债务（贪心算法）
- 重复费用（订阅）
- 离线模式
- 云同步
- 消费总额统计
- 分类费用
- 7+ 语言
- 100+ 货币
- 支付集成（Venmo/PayPal）
- 无限费用
- 无广告

#### Pro 功能（$34.99/年）
- 交易导入（银行账单）
- 货币转换
- 收据扫描（OCR）
- 项目化
- 图表
- 费用搜索
- 保存默认分摊

#### 优点
- 功能成熟、覆盖全面
- 多币种、多语言
- 国际化做得好

#### 缺点
- ⚠️ **中文支持差**（关键！）
- ⚠️ 界面偏西方化
- ⚠️ 没专门针对"旅行"场景
- ⚠️ 收费版 $34.99/年偏贵

### 2.2 百事 AA 记账（国内同类）

- **平台**: Android
- **核心功能**: AA 记账、**AI 语音识别**
- **下载量**: Google Play 100 万+
- **评分**: 4.3

#### 优点
- **AI 语音记账**（国内创新点）
- 多账本、共享
- 中文友好

#### 缺点
- ⚠️ 功能堆砌，UI 复杂
- ⚠️ 大量广告
- ⚠️ 没明确场景定位
- ⚠️ 隐私政策模糊

### 2.3 鲨鱼记账（个人记账老大）

- **平台**: iOS / Android
- **评分**: 4.6
- **特点**: 简洁、用户量大

#### 缺点
- ❌ **不是多人 AA**，是个人记账
- ❌ 不适合我们做直接参考

### 2.4 圈子账本（圈子概念）

- **平台**: iOS / Android
- **下载量**: 50 万+
- **评分**: 4.0
- **特点**: 圈子概念、多人协作

#### 缺点
- 主要面向小团队
- 旅行场景不强

### 2.5 Tricount（国际新秀）

- **平台**: iOS / Android / Web
- **用户量**: 500 万+
- **特点**: 极简、上手快、跨平台

#### 优点
- 界面极简，新用户 5 分钟上手
- 跨平台同步流畅
- 多币种支持

#### 缺点
- 国内无本地化
- 没有 AI 能力

## 四、可借鉴的具体功能

### 4.1 🚨 P0 必借鉴（已纳入 MVP v0.3）

#### 1. **语音记账**（百事 AA 杀手锏）
> 🎤 "刚才加油 200" → 自动识别 → 自动分类 + 自动选付款人
- **借鉴**：百事 AA + 来福记账
- **实现**：Android 系统 STT + 本地 LLM
- **Epic**: E-008
- **状态**: ✅ 已纳入 MVP

#### 2. **重复费用**（Splitwise Pro 核心付费点）
> 民宿每周一结、自驾保险按月
- **借鉴**：Splitwise Recurring Expenses
- **实现**：recurring_expenses 表 + workmanager 定时任务
- **Epic**: E-009
- **状态**: ✅ 已纳入 MVP

#### 3. **旅程统计图表**（国内记账 APP 标配）
> 分类饼图 + 人均柱状 + 趋势折线
- **借鉴**：Splitwise Pro Charts + 鲨鱼/钱迹/有鱼
- **实现**：fl_chart + 离线渲染
- **Epic**: E-010
- **状态**: ✅ 已纳入 MVP

### 4.2 🟡 P1 强烈建议（V1.1 评估）

#### 4. **收据 OCR**（拍照自动识别金额）
- **借鉴**：Splitwise Receipt Scanner
- **实现**：腾讯云/百度 OCR API

#### 5. **AI 智能分类**（已有雏形）
- **借鉴**：来福记账、有鱼记账
- **实现**：本地 LLM 做 NLU（语音记账同源）

#### 6. **旅程统计报告**（结算后自动出报告）
- **借鉴**：Luca 记账

### 4.3 🟢 P2 评估（V2.0 决策）

#### 7. **银行账单导入**
- **借鉴**：Splitwise Pro
- **风险**：合规成本高，国内信用卡 API 不开放

#### 8. **对话式记账**
> "我们中午吃了什么" → AI 问 → "大盘鸡 138" → 自动分摊
- **借鉴**：叨叨记账

#### 9. **支付集成**（微信支付分账）
- **借鉴**：Splitwise + Venmo
- **风险**：合规 + 商务谈判

## 五、不建议借鉴的红线（保持差异化）

| 功能 | 不借鉴原因 |
|---|---|
| 泛记账（个人记账） | 偏离"旅行场景"定位 |
| 复杂投资理财 | 偏离"记账"定位 |
| 社区/动态/朋友圈 | 偏离"工具"定位 |
| 大杂烩功能堆砌 | 与"极简自驾游"定位冲突 |
| 强制注册 | 我们是"昵称即可加入" |
| 广告变现 | 我们是"订阅变现" |
| 收费版基础功能阉割 | 我们"基础功能完整可用" |

## 六、我们的差异化定位（4 大护城河）

### 6.1 短期护城河（6-12 月）

#### 🏕️ 场景化（最强）
- **只做** 自驾游/团队游场景
- **不做** 泛记账
- 用户一看就知道"哦，这是出去玩用的"

#### 🇨🇳 中文优化（次强）
- **命名**：自驾专属分类（油费、过路费、停车费）
- **提示**：口语化文案（"没吃饭的不算"）
- **文化**：支持红包、AA 等本土习惯
- **方言**：V1.1 粤语/四川话

### 6.2 中期护城河（12-24 月）

#### 🚀 语音 + AI（追赶百事 AA）
- 语音记账 + 本地 LLM NLU
- 3 秒内一句话完成记账

#### 📊 数据沉淀（独特）
- 旅程完整体验（创建 → 记账 → 统计 → 结算 → 报告）
- 形成"出行账本"，是用户旅行回忆一部分

### 6.3 长期护城河（24+ 月）

#### 👥 关系链（社交资产）
- 多次旅行同行的"圈子"
- 切换成本随关系链累积

#### 🏢 企业版（V2.0）
- 团建、企业差旅
- 报销审批、API 集成

## 七、市场机会与风险

### 7.1 机会

| 机会 | 强度 |
|---|---|
| 中国自驾游人群 4 亿+ | ⭐⭐⭐⭐⭐ |
| 算账扯皮是公认痛点 | ⭐⭐⭐⭐⭐ |
| 没人做"自驾游专门"记账 | ⭐⭐⭐⭐⭐ |
| 旅行类 App 用户付费率高于平均 | ⭐⭐⭐⭐ |
| AI 趋势红利（语音/智能分类）| ⭐⭐⭐⭐ |

### 7.2 风险

| 风险 | 应对 |
|---|---|
| 创始人编程经验有限 | AI 工具补足（Trae + Qwen3.6） |
| 后端选型有依赖风险 | 关键逻辑本地兜底（SQLite/Hive） |
| 商业模式待验证 | 内测期收集用户付费意愿 |
| 大厂入局 | 速度 + 场景深耕 |
| 用户隐私 | 本地优先 + 透明政策 |
