## 1. 产品概述

### 1.1 一句话定位
> 专门为自驾游/团队游场景设计的智能记账与分摊工具，让多人 AA 结算从 30 分钟缩短到 30 秒。

### 1.2 产品愿景
> 让每一次团队出行都"账目清晰、关系和谐"。

### 1.3 目标市场
- **主市场**: 中国大陆 Android 用户（5 亿+）
- **次市场**: iOS 用户、境外中文用户

### 1.4 商业模式
| 版本 | 价格 | 功能 |
|---|---|---|
| 免费版 | ¥0 | 基础功能：3 个活跃旅程、15 人上限、均摊+结算 |
| 高级版 | ¥18/月 或 ¥168/年 | 无限旅程、复杂分摊、OCR、多币种、优先客服 |
| 企业版 (V2) | 联系销售 | 组织/部门管理、报销审批、API 集成 |

## 3. 核心功能 (MVP - P0)

### 3.1 旅程管理 (E-001)

**用户故事**:
- 作为组织者，我希望创建一次新旅程并添加所有成员
- 作为成员，我希望被识别身份（昵称/头像）
- 作为组织者，我希望**把成员按"组"分组**（如家庭/部门），方便按组结算

**验收标准**:
- [ ] 可创建旅程（名称、起止日期、目的地）
- [ ] 可添加/删除/修改成员（昵称即可，不需要注册）
- [ ] 可邀请成员加入（分享链接/二维码）
- [ ] 可结束/归档旅程
- [ ] 旅程首页显示基本信息 + 实时总花费
- [ ] 🆕 可创建"组"（家庭/企业/部门/团队/其他）并设置类型
- [ ] 🆕 可把成员归入一个组（一个成员最多属于一个组）
- [ ] 🆕 组可设置名称、颜色、类型

### 3.2 快速记账 (E-002)

**用户故事**:
- 作为付款人，我希望快速记录我付的某笔费用

**验收标准**:
- [ ] 3 步内完成一笔记账（付款人→类型→金额）
- [ ] 默认付款人为上次付款人
- [ ] 支持自定义备注
- [ ] 支持时间选择（默认当前）
- [ ] 支持照片/小票附件（最多 3 张）
- [ ] 列表展示所有账目（按时间倒序）
- [ ] 支持删除/编辑账目

### 3.3 基础分摊 (E-003)

**用户故事**:
- 作为组织者，我希望设置每笔账的分摊方式
- 作为组织者，我希望**按"组"分摊**（如"只算张家和李家"），而不是选一堆人

**验收标准**:
- [ ] 支持全员均摊
- [ ] 支持按比例分摊（手动设置比例）
- [ ] 支持部分人参与（指定不参与的人）
- [ ] 默认推荐均摊
- [ ] 显示每个成员应分摊金额
- [ ] 🆕 支持**按组分摊**（指定参与的是哪些组）
- [ ] 🆕 同一组内默认均摊，可调整

### 3.4 结算引擎 (E-004)

**用户故事**:
- 作为组织者，我希望自动算出"谁给谁多少钱"
- 作为组织者，我希望**按"组"维度查看结算**（如"张家应收 800 元"），而不是看个人

**验收标准**:
- [ ] 自动计算每人净收支
- [ ] 输出最优转账路径（最少笔数）
- [ ] 生成结算单
- [ ] 标记已结算项
- [ ] 显示总金额、人均、最高/最低
- [ ] 🆕 结算单支持**按组聚合**（如"张家 +800，李家 -500"）
- [ ] 🆕 组内展开可见个人明细
- [ ] 🆕 转账路径在"按组"和"按个人"两个粒度都能展示

### 3.5 🆕 语音记账 (E-008) — v0.3 新增

**用户故事**:
- 作为付款人，我希望**一句话完成记账**（"刚才加油 200"），免去手动选类别、输金额的步骤
- 作为驾驶者，我希望**单手、不看屏幕**也能记录费用

**验收标准**:
- [ ] 一句话识别：付款人 + 类别 + 金额 + 可选备注
- [ ] 识别准确率 > 90%（中文）
- [ ] 离线可用（基于 Android 系统 STT）
- [ ] 自动归类（基于关键词 + 本地 LLM Qwen3.6 35B）
- [ ] 3 秒内完成记账
- [ ] 识别结果预览 + 确认后才入库
- [ ] 支持方言识别（普通话 + 粤语 + 四川话）

### 3.6 🆕 重复费用 (E-009) — v0.3 新增

**用户故事**:
- 作为长期旅程组织者，我希望**每周的住宿/油费自动记账**，不用每周手动重复录
- 作为民宿老板/包车司机，我希望**按月结算**长期合作伙伴

**验收标准**:
- [ ] 支持固定周期：每天/每周/每月/每年
- [ ] 支持自定义日期触发（每月 15 号发工资）
- [ ] 一次性设置，长期生效
- [ ] 可暂停/恢复
- [ ] 可编辑金额和周期
- [ ] 提前 1 天通知（可选）
- [ ] 历史生成记录可查
- [ ] 智能跳过节假日（可选）

### 3.7 🆕 旅程统计图表 (E-010) — v0.3 新增

**用户故事**:
- 作为旅程成员，我希望**看到本旅程的消费图表**，了解钱花在哪了
- 作为组织者，我希望**旅程结束自动出统计报告**

**验收标准**:
- [ ] 分类饼图（油费/餐饮/住宿各占多少 %）
- [ ] 人均柱状图（谁花得多）
- [ ] 每日趋势折线
- [ ] 总金额 + 人均 + 最高/最低消费
- [ ] 一屏可看，无需滚动
- [ ] 支持分享为图片

## 4. 非功能性需求

### 4.1 性能
| 指标 | 目标 |
|---|---|
| 启动时间 | < 2 秒 |
| 记账响应 | < 500ms |
| 100 笔账目结算 | < 1 秒 |
| APK 体积 | < 30MB |

### 4.2 可用性
- 离线记账（网络恢复后同步）
- 弱网/无网场景正常使用
- 一只手操作友好
- 暗色模式支持

### 4.3 兼容性
- Android 8.0+ (API 26+)
- iOS 14.0+ (V1.1)
- 屏幕 4.7" - 6.7" 自适应

### 4.4 安全
- 用户数据加密存储（本地 SQLCipher）
- HTTPS 强制传输
- 符合《个人信息保护法》
- 不收集非必要信息

### 4.5 国际化
- MVP: 仅简体中文
- V1.1: 繁体中文、英文

## 5. 验收与发布标准

### 5.1 MVP 内测版 (M1) 发布标准
- [ ] P0 全部功能完成
- [ ] 单元测试覆盖率 > 60%
- [ ] 至少 5 个内测用户完成完整旅程
- [ ] 无 P0/P1 级别 bug
- [ ] 内测版本成功运行 2 周

### 5.2 1.0 正式版发布标准
- [ ] P0 + P1 功能完成
- [ ] 用户反馈迭代 ≥ 3 轮
- [ ] 通过 Google Play 审核
- [ ] 应用商店评分 ≥ 4.0

### 5.3 关键质量指标
- 7 日留存率 ≥ 30%
- 旅程完成率（创建→记账→结算）≥ 50%
- 崩溃率 < 0.5%
