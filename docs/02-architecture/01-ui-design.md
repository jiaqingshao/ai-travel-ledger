## 设计原则

| 原则 | 体现 |
|---|---|
| **3 秒记账** | 大按钮 + 默认值 + 数字键盘 |
| **中文友好** | 货币 ¥、自驾专属分类、口语化文案 |
| **一只手操作** | FAB 在右下、按钮在底部、滑动操作 |
| **Material Design 3** | 圆角 16dp、动态颜色、ripple 反馈 |
| **暗色模式优先** | OLED 友好、长途夜车不刺眼 |

## 真实 UI 截图生成方案

由于 PDF/Chrome/wkhtmltopdf 都不可用，**生成真实 UI 截图有 3 种方案**：

### 方案 A：Flutter Widget 截图（推荐）

写一个 Flutter 应用，把每个屏幕用真实 Widget 渲染，调用 `RepaintBoundary.toImage()` 导出 PNG。

```dart
// scripts/ui_preview.dart
import 'package:flutter/material.dart';

void main() => runApp(UIPreviewApp());

class UIPreviewApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(home: TripListScreenMock());
  }
}

// 每个屏幕一个 mock Widget
class TripListScreenMock extends StatelessWidget { ... }

// 截图脚本
Future<void> captureScreen(Widget screen, String filename) async {
  final repaintBoundary = RepaintBoundary(child: screen);
  // ... 渲染 + toImage + 保存 PNG
}
```

**优点**：
- ✅ 真实 Material 3 渲染
- ✅ 包含暗色模式
- ✅ 不同屏幕尺寸（4.7" / 6.4" / 6.7"）
- ✅ 一键批量生成

**缺点**：
- ⚠️ 需要 Flutter 环境（已就绪）
- ⚠️ 工作量：约 2 小时

### 方案 B：HTML + Puppeteer（次选）

把线框图用 HTML/CSS 重写，用 Chromium headless 截图。

**优点**：
- ✅ 真实 CSS 渲染
- ✅ 跨平台

**缺点**：
- ⚠️ Puppeteer 安装复杂
- ⚠️ 不是真 Material 3

### 方案 C：image_generate 工具（最快）

用我现有的图像生成工具直接画 UI 草图。

**优点**：
- ✅ 立刻生成
- ✅ 美观

**缺点**：
- ⚠️ 不是真实代码渲染
- ⚠️ 仅供设计参考
