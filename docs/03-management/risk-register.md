## 风险详情

### R001 - 创始人编程经验不足 ⚠️ 极高

**描述**:
- 创始人有 C、Python 基础
- 无 Flutter / Dart / 移动开发经验
- 无后端 / 数据库 / DevOps 经验

**影响**:
- 进度可能严重延迟
- 代码质量可能不高
- 架构决策可能走弯路

**应对措施**:
1. **AI 工具最大化** - Trae AI Builder 模式 + Composer Agent
2. **本地模型** - Qwen3.6 35B 几乎 0 token 费
3. **外部支持** - 需要时雇 Flutter 兼职做 Code Review
4. **分阶段** - 先 MVP，不追求完美架构
5. **记录决策** - 所有技术选型写 ADR，未来不返工

**状态**: 监控中，持续到 MVP 发布

### R002 - 项目周期延长 ⚠️ 高

**描述**:
- 学习 + 开发双线作战
- 容易陷入"完美主义"

**应对措施**:
- 严格 MVP 范围，P0 优先
- 2 周一次复盘
- 砍需求原则：用户说"想要"≠ "必须做"

**关键里程碑**:
- T+2 月 MVP 内测
- T+3 月 1.0 上架
- 超期 > 1 月：触发需求砍剪

### R010 - 个人精力分散 ⚠️ 高

**描述**:
- 创始人可能同时被本职 / 家庭 / 其他项目分心

**应对措施**:
- 周计划 / 月计划明确
- 每天固定时间块
- 风险登记定期 review
