## 🔍 问题与解决

### P1 隐患：上次会话 dangling state

- **症状**：发现 4 文件改 + 3 文件新未入仓（来自前一次会话的 ISSUE-027~030 修复）
- **根因**：agent 干完一段没立即 commit，~200 行代码游离在 git 之外
- **解决**：commit `6c4d04b` 把运行时改造 + in-app UI + 测试同步一起入仓（11 files, +758 / -22）
- **教训 (L4)**：每次干完一段必须 commit，不能留 dangling state

### P1 编译错误：currentSession 链式访问错位

- **症状**：`flutter test` 编译失败：`Error: The getter 'currentSession' isn't defined for the class 'User'`
- **根因**：`auth.currentUser?.currentSession?.user.email` — `currentSession` 在 `GoTrueClient` 上，不在 `User` 上
- **解决**：改为 `auth.currentSession?.user.email`
- **教训**：链式访问 API 前要查文档，不能凭直觉

### P1 编译错误：DemoSeed.apply 缺参数（release build 才发现）

- **症状**：`flutter build apk --release` 失败：`Required named parameter 'appSettings' must be provided`
- **根因**：6c4d04b 给 `HiveBoxes` 加 required `appSettings`，改了 3 个测试文件，**但漏改 `lib/main.dart`**
- **为什么 test 没发现**：`flutter test` 不触发 `SEED=demo` 分支，只有真 release build 走
- **解决**：commit `5b51a9d` 加 1 行 `DemoSeed.apply(context, appSettings: appSettings)`
- **教训 (L5)**：**发版前必须跑一次 release build，光跑 test 不够**，test 覆盖路径 ≠ 真生产路径

### P2 文档损坏：issue-tracker.md 混合编码

- **症状**：PowerShell `Get-Content` 显示乱码（中文 ISSUE 详情变 ???）
- **根因**：文件开头是 UTF-8 BOM，但中间段 ISSUE-016~024 是 GBK 字节被错误嵌入
- **解决**：用 Python 一次性重写整份为干净 UTF-8
- **教训**：未来编辑 issue-tracker 必须保持统一编码（建议全用 Python/Node 写）
